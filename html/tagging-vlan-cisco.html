TAGGING VLAN CISCO NEXUS DAN QinQ

Untuk membuat VLAN dan melakukan tagging pada switch Cisco Nexus, Anda bisa mengikuti langkah-langkah berikut. Pastikan Anda memiliki akses ke perangkat dan hak istimewa yang diperlukan untuk melakukan konfigurasi.

1. Masuk ke Mode Konfigurasi

Pertama, masuk ke switch Anda melalui SSH atau konsol, lalu masuk ke mode konfigurasi.

enable

configure terminal

2. Membuat VLAN

Gunakan perintah berikut untuk membuat VLAN baru. Misalnya, untuk membuat VLAN 10:

vlan 10

  name VLAN10

3. Mengonfigurasi Interface

Setelah VLAN dibuat, Anda perlu mengonfigurasi interface yang akan terhubung ke VLAN tersebut. Anda bisa mengonfigurasi interface akses atau trunk.

a. Mengonfigurasi Interface Akses

Jika Anda ingin mengonfigurasi interface sebagai akses untuk VLAN tertentu:

interface ethernet 1/1

  switchport mode access

  switchport access vlan 10

b. Mengonfigurasi Interface Trunk

Jika Anda ingin mengonfigurasi interface sebagai trunk (untuk mengizinkan beberapa VLAN):

interface ethernet 1/2

  switchport mode trunk

  switchport trunk allowed vlan 10

4. Mengonfigurasi Tagging

Dalam konfigurasi trunk, VLAN akan ditandai (tagged) secara otomatis. Namun, jika Anda ingin memastikan VLAN yang diizinkan di trunk, gunakan perintah berikut:

switchport trunk allowed vlan add 10

5. Menyimpan Konfigurasi

Setelah selesai melakukan konfigurasi, jangan lupa untuk menyimpan perubahan:

end

write memory

6. Memverifikasi Konfigurasi

Anda bisa memverifikasi konfigurasi VLAN dan interface dengan perintah berikut:

show vlan brief

show interface trunk

Contoh Lengkap

Berikut adalah contoh lengkap dari langkah-langkah di atas:

enable

configure terminal

 

vlan 10

  name VLAN10

 

interface ethernet 1/1

  switchport mode access

  switchport access vlan 10

 

interface ethernet 1/2

  switchport mode trunk

  switchport trunk allowed vlan 10

 

end

write memory

Dengan langkah-langkah di atas, Anda telah berhasil membuat VLAN dan mengonfigurasi tagging pada switch Cisco Nexus.

 

QinQ (802.1Q tunneling) adalah metode untuk mengenkapsulasi VLAN dalam VLAN yang memungkinkan pengangkutan banyak VLAN melalui jaringan tanpa kehilangan informasi VLAN asli. Pada switch Cisco Nexus, Anda dapat mengonfigurasi QinQ dengan mengikuti langkah-langkah di bawah ini.

Langkah-langkah untuk Mengonfigurasi QinQ

1. Masuk ke Mode Konfigurasi

Pertama, Anda perlu terhubung ke switch melalui SSH atau konsol. Setelah terhubung, masuk ke mode konfigurasi:

bash

Copy code

enable

configure terminal

 

2. Membuat VLAN

Buat dua VLAN, satu untuk VLAN luar (S-VLAN) dan satu untuk VLAN dalam (C-VLAN). Misalnya, untuk VLAN 100 sebagai S-VLAN dan VLAN 10 sebagai C-VLAN:

bash

Copy code

vlan 100

  name S-VLAN

 

vlan 10

  name C-VLAN

3. Mengonfigurasi Interface Trunk

Pilih interface yang akan digunakan untuk QinQ dan atur mode trunk. Misalnya, kita akan menggunakan interface Ethernet 1/1:

bash

Copy code

interface ethernet 1/1

  switchport mode trunk

  switchport trunk allowed vlan 10,100

  switchport trunk native vlan 100

  switchport mode dot1q-tunnel

  • switchport mode dot1q-tunnel: Perintah ini mengatur interface untuk menggunakan mode QinQ.

 

 

4. Mengonfigurasi VLAN pada Interface

Setelah interface diatur ke mode QinQ, Anda perlu mengonfigurasi VLAN yang akan digunakan untuk QinQ:

interface ethernet 1/1

  switchport qinq enable

5. Menyimpan Konfigurasi

Setelah selesai mengonfigurasi QinQ, jangan lupa untuk menyimpan konfigurasi:

end

write memory

6. Memverifikasi Konfigurasi

Untuk memverifikasi bahwa QinQ telah dikonfigurasi dengan benar, Anda dapat menggunakan perintah berikut:

show vlan brief

show interface trunk

show running-config interface ethernet 1/1

Contoh Lengkap

Berikut adalah contoh lengkap dari langkah-langkah di atas:

configure terminal

 

vlan 100

  name S-VLAN

 

vlan 10

  name C-VLAN

 

interface ethernet 1/1

  switchport mode trunk

  switchport trunk allowed vlan 10,100

  switchport trunk native vlan 100

  switchport mode dot1q-tunnel

  switchport qinq enable

 

end

write memory

Dengan langkah-langkah di atas, Anda telah berhasil mengonfigurasi QinQ pada switch Cisco Nexus.
